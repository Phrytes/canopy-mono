# `@onderling/app-scaffold` — API reference

> Generated by `node scripts/api-appendix.mjs` from the exported surface and source JSDoc.
> Do not edit by hand — edit the JSDoc in the source and regenerate. `npm run api-fitness` fails on drift.

SP-10 v0 — a pure manifest→app scaffolder. Takes an app manifest + a `requires` capability list (SP-9 vocabulary), validates the requires, and returns a runnable app skeleton (package.json + createAgent entry + one wireSkill stub per manifest operation) as path→content strings. PURE codegen — a `writer` may be injected to also write to disk, but the core is string-returning + testable.

### `CAPABILITY_SDK_IMPORT`

**Kind:** constant · **Import:** `CAPABILITY_SDK_IMPORT` from `'@onderling/app-scaffold'`

SP-9 capability → the SDK import specifier the scaffolded app should use.
Every specifier resolves to the single `@onderling/sdk` package (sub-path
exports), so the npm dependency is always just `@onderling/sdk`; this map is
what the generated code IMPORTS and what `package.json.canopy.sdkImports`
records per requested capability.

  - core       → @onderling/sdk/core
  - transports → @onderling/sdk/transports
  - vault      → @onderling/sdk/vault
  - pod        → @onderling/sdk/pod
  - high       → @onderling/sdk   (the barrel re-exports the /high slice, so the
                                opinionated helpers import from the barrel)
