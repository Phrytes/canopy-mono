# `@onderling/logger` — API reference

> Generated by `node scripts/api-appendix.mjs` from the exported surface and source JSDoc.

Privacy-first structured logging facade + on-device ring buffer (web ≡ mobile). PII-safe by construction: events are a tag + a stable code + sanitized scalar fields — never free-text content or identities.

README: [`packages/logger/README.md`](../../packages/logger/README.md) · Index: [docs/api/README.md](README.md)

### `log`

**Kind:** constant · **Import:** `log` from `'@onderling/logger'`

The logging facade. Usage: `log.info('feedback', 'consent.stored', { n: 2 })`. No free-text message param.

### `dumpLogs`

**Kind:** function · **Import:** `dumpLogs` from `'@onderling/logger'`

The recent records (shallow copy) — for a bug report / a "copy logs" affordance.

### `formatLogs`

**Kind:** function · **Import:** `formatLogs` from `'@onderling/logger'`

```js

```

One line per record, PII-safe (codes + scalar fields only). Ready to show the user / copy to clipboard.

### `clearLogs`

**Kind:** function · **Import:** `clearLogs` from `'@onderling/logger'`

Empty the log ring buffer in place.

### `configureLog`

**Kind:** function · **Import:** `configureLog` from `'@onderling/logger'`


Configure the logger (dev sink, min level, ring size, injectable clock for tests).

### `consoleSink`

**Kind:** function · **Import:** `consoleSink` from `'@onderling/logger'`


A dev sink that mirrors records to the console (attach in dev builds; NEVER in prod).
