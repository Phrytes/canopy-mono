# `@onderling/redaction` — API reference

> Generated by `node scripts/api-appendix.mjs` from the exported surface and source JSDoc.
> Do not edit by hand — edit the JSDoc in the source and regenerate. `npm run api-fitness` fails on drift.

Generic, config-driven redaction atom. redact(text, config) → {text, hits}: ordered regex rules + named-validator registry (bsn-11proef, nl-phone, iban, luhn) + an optional gazetteer name-pass. Locale-agnostic — all locale content is data the caller supplies. Extracted from feedback-pipeline's redact.js + names.js.

## `src/validators.js`

### `bsn11proef`

**Kind:** function · **Import:** `bsn11proef` from `'@onderling/redaction'`, `'@onderling/redaction/validators'`

```js
bsn11proef(s)
```

Dutch BSN (burgerservicenummer) 11-proef checksum over exactly 9 digits.
Separators are NOT stripped here — pass the bare digit run (the rule's pattern
decides what reaches the validator). ~10/11 of random 9-digit numbers fail.

**Parameters**

- `s` `string`

**Returns:** `boolean`

### `nlPhone`

**Kind:** function · **Import:** `nlPhone` from `'@onderling/redaction'`, `'@onderling/redaction/validators'`

```js
nlPhone(s)
```

Dutch phone number, validated by digit-count + prefix after stripping all
non-digits from the candidate. Accepts 06xxxxxxxx / 0xx-xxxxxxx, 31 6xxxxxxxx,
and the typo-tolerant +31 0… form.

**Parameters**

- `s` `string` — raw candidate (may contain spaces / () . -)

**Returns:** `boolean`

### `luhn`

**Kind:** function · **Import:** `luhn` from `'@onderling/redaction'`, `'@onderling/redaction/validators'`

```js
luhn(s)
```

Luhn (mod-10) checksum — the check used by payment-card numbers and many other
identifier schemes. Strips non-digits before validating.

**Parameters**

- `s` `string`

**Returns:** `boolean`

### `iban`

**Kind:** function · **Import:** `iban` from `'@onderling/redaction'`, `'@onderling/redaction/validators'`

```js
iban(s)
```

IBAN check-digit validation (ISO 7064 mod-97-10). Strips spaces, requires the
2-letter country + 2 check-digit head, moves the first 4 chars to the end,
converts letters to numbers (A=10 … Z=35) and checks mod 97 === 1.

**Parameters**

- `s` `string`

**Returns:** `boolean`

### `VALIDATORS`

**Kind:** constant · **Import:** `VALIDATORS` from `'@onderling/redaction'`, `'@onderling/redaction/validators'`

The named-validator registry: maps a rule's `validate` name ('bsn-11proef', 'nl-phone', 'iban',
'luhn') to its pure predicate. The redaction engine only redacts a candidate match when the rule's
named validator returns true.
