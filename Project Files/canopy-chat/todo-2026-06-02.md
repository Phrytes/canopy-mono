## 6 · `chatClaudeDesign` extraction (2026-06-02)

Ideas mined from the root `chatClaudeDesign` file (a v3 design conversation, NL).
The 7 framing questions all got Frits answers; below = each idea, its build state,
and where it merges with this todo.  Most map onto already-shipped P6.* / 5.* /
Slice-6 work; the genuinely under-served items are marked **NEW**.

### Decisions Frits gave in that chat (now canon)
- **Q1 chat ownership:** individual can ALWAYS turn chat off (others may get a
  notice if they try anyway); a kring admin can turn kring-chat off (consensus per
  admin settings); AND a user can build ONE cross-kring stream where everything
  lands. → chat-off ✅ (P6.4 + wederkerigheid), admin chat-off ✅ (P6.1+P6.2 consensus),
  cross-kring stream ✅ substrate (α.2 userScreens, `kringFilter: null` = all).
- **Q2 "Anne moves in":** keep as THREE inline follow-up actions (not one mega-action).
  → ✅ surfaces.followUps (CC-XA.3 tested).
- **Q3 reveal:** 1-on-1 by default, per-group override allowed but must be loud in
  onboarding. → revealPolicy axis ✅ (default 'pairwise'); **onboarding-clarity = 🟡 gap**.
- **Q4 kring vs primitive:** Archive/Folio is a PRIVATE app that can also be
  agent-linked / shared-link / group-pod-drive-accessed; PoL stands alone (future
  kring access-requirement); import-bridge dropped. These are skills-in-kringen,
  not kringen. → Folio dual-life ✅ mostly (P6.M7/M8 + 5.2); **group-pod drive-like
  access = 🟡 partial**; **PoL-as-access-gate = 🔴 future** (5.9d placeholder only).
- **Q5 roles:** minimalist admin+member default, schema allows adding roles —
  **with templates** when you want to add one. → roles ✅ (S5); **role templates = NEW**
  (folds into E8/templates).
- **Q6 LLM tool vs agent:** an agent can just be added by someone; admin gates
  whether agents are allowed; individuals can opt out of agents; LLM-as-tool is
  also optional; the two don't exclude. → agents axis + opt-out ✅; LLM-pluggability
  deferred-by-design (L4). Not well-specified yet how "add an LLM" works.
- **Q7 tier-labels:** add 🟢/🟡/🔴 per board in the v3 design doc, with a todo that
  Claude Code fills them against real repo state. → **design-doc task (NEW)**.

### NEW / under-served items worth adding
- **N1 — Buurt template = prikbord-first, chat-OFF by default, chat-per-post.**
  Frits: the buurt default should have NO open chat; a chat only appears when
  someone reacts to a vraag/aanbod. The per-post DM-spawn already exists (Slice 6b
  `/help-with`), but the `buurt` template still sets `chat:true`. **This IS the
  "more elaborate templates" idea — merge into E8:** flip the buurt template to
  noticeboard+agenda (chat off), and have reacting-to-a-post spawn the thread.
- **N2 — Info-icon (ⓘ) "consequences per option" in every choice form.**
  Frits wants each radio option in the create wizard, kring-settings, personal
  override, hop settings, and skill creation to carry an ⓘ that reveals "Gevolgen
  als je dit kiest: …".  **Not implemented anywhere today.** Cross-cutting UX, web+mobile.
- **N3 — Roles templates** (from Q5) — a starter set of roles (gast / observer /
  externe-vrijwilliger) an admin can opt a kring into; folds into the E8 template work.
- **N4 — v3 standalone design doc + tier-labels.** The chat produced
  "Canopy Interface Exploration v3.html" (11 sections) — NOT in the repo. If it's
  to be canon alongside the v2 PDF, it needs importing + Claude Code filling the
  🔘 tier-labels per board against the real repo state.
- **N5 — Group-pod drive-like Folio access** (from Q4) — Google-Drive-style access
  to a group pod's Folio, distinct from private notes + share-links. Partially via
  saveToMyPod/listFiles but no "browse a shared group drive" surface.
