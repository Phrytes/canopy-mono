## C. UX-bug followups surfaced 2026-05-24 manual test

These came up when Frits tested the Cluster A+B surfaces in the browser
and reported unreadable / unfunctional outputs.  Tracked here to fold
into the next polish slice.

- **`/crew-new` then `/crews`** doesn't show the newly-provisioned
  crew.  Cause likely: `getMyCrews`'s `crewsProvider()` bound at
  agent boot; provisionMyCrew adds a CrewState that the provider
  doesn't know about.
- **`/crew-members` `members` field** renders as raw JSON string
  (the chat-shell's `formatFieldValue` JSON-stringifies arrays of
  objects).  Should render as a nice sub-list with webid+role per
  row.
- **`/availability` empty visible output** — likely the
  disabled-for-crew branch isn't rendering, or the empty-grid case
  hits a renderer edge.  Needs reproduction + targeted fix.
- **Audit for similar patterns** — any adapter that surfaces a
  nested array/object as a record field will JSON-stringify and
  become unreadable.  Sweep the realAgent.js adapters.
